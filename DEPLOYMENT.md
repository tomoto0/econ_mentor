## デプロイメント手順

### ステップ1: チェックポイント作成

デプロイメント前に、必ずプロジェクトのチェックポイントを作成してください。

```bash
# Management UIで「Checkpoint」ボタンをクリック
# または以下のコマンドで確認
git status
git log --oneline -5
```

### ステップ2: テスト実行

本番環境へのデプロイメント前に、すべてのテストが成功していることを確認してください。

```bash
# ユニットテスト実行
pnpm test

# ビルドテスト
pnpm build

# 開発サーバーで動作確認
pnpm dev
```

### ステップ3: 環境変数確認

本番環境で使用される環境変数が正しく設定されていることを確認してください。

**必須環境変数**:
- `DATABASE_URL`: PostgreSQL接続文字列
- `JWT_SECRET`: セッション署名用シークレット
- `VITE_APP_ID`: Manus OAuth アプリケーションID
- `OAUTH_SERVER_URL`: Manus OAuth サーバーURL
- `VITE_OAUTH_PORTAL_URL`: Manus ログインポータルURL
- `BUILT_IN_FORGE_API_KEY`: Manus LLM API キー
- `VITE_FRONTEND_FORGE_API_KEY`: フロントエンド用APIキー

**オプション環境変数**:
- `VITE_APP_TITLE`: アプリケーションタイトル
- `VITE_APP_LOGO`: ロゴURL

### ステップ4: Publishボタンをクリック

1. Management UI（右側パネル）を開く
2. 最新のチェックポイントを確認
3. 「Publish」ボタンをクリック
4. デプロイメントが開始されます

### ステップ5: デプロイメント確認

デプロイメント後、以下を確認してください：

```bash
# 本番URLにアクセス
# https://[your-domain].manus.space

# ホームページが正常に表示されることを確認
# チャット機能が動作することを確認
# グラフが生成されることを確認
```

## トラブルシューティング

### デプロイメント失敗

**症状**: Publishボタンが無効化されている

**解決方法**:
1. 最新のチェックポイントを作成してください
2. すべてのテストが成功していることを確認してください
3. ビルドエラーがないことを確認してください

### 本番環境でのエラー

**症状**: 本番環境でチャット機能が動作しない

**解決方法**:
1. 環境変数が正しく設定されていることを確認
2. データベース接続が確立されていることを確認
3. LLM APIキーが有効であることを確認

### パフォーマンス問題

**症状**: ページの読み込みが遅い

**解決方法**:
1. CDNキャッシュをクリア
2. データベースインデックスを確認
3. 不要なAPIコールを削除

## ローリングアップデート

新しいバージョンをデプロイする際は、以下の手順に従ってください：

1. **開発環境でテスト**: 新機能を開発環境で十分にテスト
2. **チェックポイント作成**: 新しいチェックポイントを作成
3. **段階的デプロイ**: 必要に応じて段階的にユーザーに展開
4. **モニタリング**: デプロイ後、エラーログを監視

## ロールバック手順

問題が発生した場合、以下の手順で前のバージョンに戻すことができます：

1. Management UI の「Dashboard」を開く
2. 前のチェックポイントを確認
3. 「Rollback」ボタンをクリック
4. 確認ダイアログで「Yes」をクリック

## セキュリティチェックリスト

デプロイメント前に、以下のセキュリティチェックを実施してください：

- [ ] すべてのシークレット/APIキーが環境変数に設定されている
- [ ] データベース接続が暗号化されている（SSL/TLS）
- [ ] HTTPSが有効化されている
- [ ] CORS設定が正しく設定されている
- [ ] 認証トークンの有効期限が設定されている
- [ ] ログに機密情報が含まれていない
- [ ] 依存関係に既知の脆弱性がない

## スケーリング

EconoMentorは、Manusプラットフォームのオートスケーリング機能により、トラフィック増加に自動的に対応します。

### スケーリング設定

- **自動スケーリング**: 有効
- **最小インスタンス数**: 1
- **最大インスタンス数**: 10
- **スケールアップトリガー**: CPU使用率 > 70%
- **スケールダウントリガー**: CPU使用率 < 30%
