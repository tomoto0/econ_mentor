# EconoMentor - テストドキュメント

## テスト戦略

このドキュメントでは、EconoMentorの主要機能をテストするための手順を説明します。

## 1. 基本機能テスト

### 1.1 ホームページ表示テスト

**目的**: ホームページが正常に表示されることを確認

**手順**:
1. ブラウザで https://3000-iitlpldszayiwf1ll9uy1-05d7b414.manus.computer にアクセス
2. 以下の要素が表示されることを確認：
   - ページタイトル「AI経済学メンターと学ぶ」
   - 3つの機能カード（専門的な解説、動的グラフ、リアルタイム対話）
   - トピック入力フォーム
   - 人気トピックボタン（需要と供給、インフレーション等）

**期待される結果**: すべての要素が正常に表示される

### 1.2 セッション開始テスト

**目的**: 新しい学習セッションが正常に開始されることを確認

**手順**:
1. トピック入力フィールドに「需要と供給」と入力
2. 「学習を開始」ボタンをクリック
3. 学習ページにリダイレクトされることを確認
4. ページタイトルに「需要と供給」が表示されることを確認

**期待される結果**: 学習ページが正常に表示され、セッションが開始される

## 2. チャット機能テスト

### 2.1 メッセージ送受信テスト

**目的**: AIメンターとのメッセージ送受信が正常に機能することを確認

**手順**:
1. 学習ページで「需要と供給について説明してください」と入力
2. 送信ボタンをクリック
3. ユーザーメッセージが画面に表示されることを確認
4. AIメンターからの応答が表示されることを確認
5. 応答にマークダウンフォーマットが含まれている場合、正しく表示されることを確認

**期待される結果**: メッセージが正常に送受信され、AIの応答が表示される

### 2.2 複数メッセージテスト

**目的**: チャット履歴が正常に保存・表示されることを確認

**手順**:
1. 複数の質問を連続で送信（例：3つ以上）
2. すべてのメッセージが画面に表示されることを確認
3. メッセージの順序が正しいことを確認
4. ページを更新してもメッセージが保持されることを確認

**期待される結果**: すべてのメッセージが正常に保存・表示される

## 3. グラフ生成テスト

### 3.1 需要供給曲線テスト

**目的**: グラフ生成機能が正常に動作することを確認

**手順**:
1. 学習ページで「需要と供給曲線を描いて」と入力
2. AIメンターからの応答にグラフが含まれることを確認
3. グラフが正しく描画されることを確認（軸ラベル、タイトル、曲線）
4. グラフのインタラクティブ機能をテスト（ホバーでツールチップ表示等）

**期待される結果**: グラフが正常に生成・表示される

### 3.2 複数グラフテスト

**目的**: 複数の異なるグラフが生成できることを確認

**手順**:
1. 「費用曲線を描いて」と入力
2. グラフが表示されることを確認
3. 「フィリップス曲線を示してください」と入力
4. 異なるグラフが表示されることを確認

**期待される結果**: 複数の異なるグラフが正常に生成される

## 4. セッション管理テスト

### 4.1 セッション復元テスト

**目的**: ページを離れて戻った場合、セッションが復元されることを確認

**手順**:
1. 学習ページで複数のメッセージを送信
2. ブラウザを閉じる（または別のタブで異なるページにアクセス）
3. 再度EconoMentorにアクセス
4. 前回のセッションが復元されることを確認
5. チャット履歴がすべて表示されることを確認

**期待される結果**: セッションが正常に復元される

### 4.2 複数セッション管理テスト

**目的**: 複数のセッションを管理できることを確認

**手順**:
1. セッションAで「インフレーション」について学習
2. ホームページに戻る
3. セッションBで「GDP」について学習
4. 各セッションのチャット履歴が独立していることを確認

**期待される結果**: 複数のセッションが独立して管理される

## 5. レスポンシブデザインテスト

### 5.1 モバイルデバイステスト

**目的**: モバイルデバイスで正常に表示されることを確認

**手順**:
1. ブラウザの開発者ツールで「デバイスモード」を有効化
2. iPhone 12（390x844）で表示を確認
3. すべてのテキストが読みやすいことを確認
4. ボタンがタップ可能なサイズであることを確認
5. チャットインターフェースがスクロール可能であることを確認

**期待される結果**: モバイルデバイスで正常に表示される

### 5.2 タブレットデバイステスト

**目的**: タブレットデバイスで正常に表示されることを確認

**手順**:
1. iPad（768x1024）で表示を確認
2. グラフが適切なサイズで表示されることを確認
3. レイアウトが最適化されていることを確認

**期待される結果**: タブレットデバイスで正常に表示される

## 6. パフォーマンステスト

### 6.1 ページロード時間テスト

**目的**: ページが迅速に読み込まれることを確認

**手順**:
1. ブラウザの開発者ツールで「Network」タブを開く
2. ホームページを読み込む
3. 初回読み込み時間が3秒以内であることを確認
4. 学習ページの読み込み時間が2秒以内であることを確認

**期待される結果**: ページが迅速に読み込まれる

### 6.2 チャット応答時間テスト

**目的**: AIメンターの応答が適切な時間内に返される

**手順**:
1. メッセージを送信
2. 応答が10秒以内に返されることを確認
3. 複数のメッセージを連続送信
4. すべてのメッセージが処理されることを確認

**期待される結果**: AIメンターが適切な時間内に応答する

## 7. エラーハンドリングテスト

### 7.1 無効な入力テスト

**目的**: 無効な入力が適切に処理されることを確認

**手順**:
1. 空のメッセージを送信しようとする
2. エラーメッセージが表示されることを確認
3. 非常に長いメッセージを送信
4. 正常に処理されることを確認

**期待される結果**: 無効な入力が適切に処理される

### 7.2 ネットワークエラーテスト

**目的**: ネットワークエラーが適切に処理されることを確認

**手順**:
1. ブラウザの開発者ツールでネットワークを「Offline」に設定
2. メッセージを送信しようとする
3. エラーメッセージが表示されることを確認
4. ネットワークを復旧
5. メッセージが正常に送信されることを確認

**期待される結果**: ネットワークエラーが適切に処理される
